Compromiso de accesibilidad

Este sitio web se ha diseñado y desarrollado para ser accesible y usable para todas las personas, incluyendo aquellas con discapacidad visual, auditiva, física, cognitiva o neurológica. Se siguen las pautas del WCAG 2.1 nivel AA, garantizando que los errores detectables sean corregidos y documentados.

Identificación del idioma principal

El idioma principal de este sitio web es el español (<html lang="es">), asegurando la correcta lectura por lectores de pantalla y motores de traducción. Para contenido en otros idiomas se utilizan los atributos adecuados, por ejemplo <span lang="en">.

Normativa aplicada

Esta declaración se realiza conforme a la Decisión de Ejecución (UE) 2018/1523 de la Comisión, de 12 de octubre de 2018, sobre la accesibilidad de los sitios web y aplicaciones móviles de los organismos del sector público. Además, se siguen las pautas WCAG 2.1 nivel AA.

Correcciones tras revisión automática

Se evaluó el sitio con herramientas automáticas (WAVE, axe DevTools, Google Lighthouse), detectando los siguientes problemas:

  • Contraste insuficiente en texto, enlaces y botones.
  • Enlaces no distinguibles del texto circundante solo por color.
  • Errores de estructura en tablas y encabezados.

Tras ajustar colores en CSS y corregir estilos, los análisis posteriores mostraron 0 errores, cumpliendo con WCAG 2.1 nivel AA.

Correcciones tras revisión manual

Se realizaron revisiones manuales usando herramientas como Accessibility Inspector, HeadingsMap, Colour Contrast Analyser y ARC Toolkit, detectando y corrigiendo:

  • Checkbox sin etiqueta accesible: Añadidos <fieldset> y <legend> para agrupar opciones.
  • Spinbutton no accesible con teclado: Modificado para poder seleccionar la fecha solo con teclado.
  • Landmark mal ubicado o sin encabezado: Se corrigió la ubicación de <aside> fuera del <main>.
  • Saltos incorrectos en la jerarquía de encabezados: Se ajustó la secuencia de <h1> a <h6> para mantener coherencia semántica.
  • Contraste insuficiente entre texto y fondo: Ajustados colores según WCAG 2.1 AA.
  • Botones con contraste insuficiente: Modificados para cumplir con ratio mínimo 4.5:1.
  • Botón sin id ni label: Se añadió id y aria-label para mejorar accesibilidad.
  • <iframe> sin title: Se añadieron atributos title descriptivos.
  • <iframe> sin lang definido: Documentado como limitación técnica por contenido externo.

Etiquetado semántico

Se utiliza HTML5 semántico para estructurar la página, incluyendo:

  • <header> para la cabecera.
  • <nav> para la navegación principal.
  • <main> para el contenido principal.
  • <aside> para información complementaria.
  • <article> y <section> para organizar el contenido en secciones y subsecciones.
  • <footer> para el pie de página.

Texto alternativo en imágenes

Todas las imágenes contienen atributos alt que describen su función o contenido. Las imágenes decorativas usan alt="". Para imágenes complejas se emplean descripciones detalladas (longdesc si fuera necesario).

Mecanismos de navegación

El sitio incluye múltiples mecanismos para facilitar la navegación:

  • Mapa del sitio textual y visual.
  • Migas de pan (breadcrumbs) que muestran la ruta desde la página principal.
  • Enlaces claros y agrupados en listas para facilitar la localización de contenido.
  • Navegación completamente funcional mediante teclado.

Otras características de accesibilidad implementadas

  • Contraste adecuado entre texto y fondo, verificado con herramientas de comprobación.
  • Textos fragmentados en párrafos cortos y secciones claras para mejorar la comprensión.
  • Formularios accesibles con etiquetas <label>, agrupados en <fieldset> y <legend>.
  • Evitar la transmisión de información solo mediante color o elementos visuales.
  • Consistencia en estructura y navegación en todas las páginas.

Conformidad y limitaciones

Este sitio web es parcialmente conforme con el nivel AA de las WCAG 2.1, debido a algunas excepciones técnicas y contenido de terceros que todavía no cumple completamente con todos los criterios de accesibilidad.
